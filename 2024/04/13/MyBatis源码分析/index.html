MyBitis源码分析

发表于2024-04-13|更新于2024-04-13
|字数总计:4.7k|阅读时长:21分钟|阅读量:

MyBatis框架分析概要

  • 框架就是xml文件解析+反射机制+设计模式
  • 核心类就是SqlSessionFactoryBuilder和SqlSession

第一步 从使用者角度出发

  • 核心配置文件godbatis.xml
  • mapper映射文件SqlMapper.xml
  • 测试类

第二步 创建Resources工具类

  • 根据测试类中Resources.getResourceAsStream(“godbatis-config.xml”),这是从根路径中加载xml文件,返回值应该是个文件输入流InputStream

第三步 创建Const常量类

  • 方便参数的修改

第三步 创建SqlSessionFactoryBuilder类

  • 根据测试类中SqlSessionFactoryBuilder sqlSessionFactoryBuilder = new SqlSessionFactoryBuilder();,这个对象应该具有build方法,根据文件输入流获取到一个SqlSessionFactory对象
  • build方法主要是用dom4j进行xml文件解析,在这个方法里还涉及了多个对象,分别是数据源对象DataSource,事务管理器对象Transaction,以及用于封装sql语句id与sql内容(自定义的MappedStatement对象)的mappedStatements对象
  • factory = new SqlSessionFactory(transaction, mappedStatements);由这行代码可知,想构建出一个SqlSessionFactory对象还需要两个参数,事务管理器和自定义sql信息集合
  • private DataSource getDataSource(Element dataSourceElt)方法主要获取到dom4j文件解析的数据源部分的数据,然后获取到配置好的数据源对象
  • private Transaction getTransaction(Element transactionManagerElt, DataSource dataSource) 方法主要获取到dom4j文件解析的事务管理器部分的数据,以及数据源,然后获取到配置好的事务管理器对象
  • 下面的步骤(四到七)就用来具体实现这些类的创建

第四步 创建sql封装MappedStatement类

  • 这个类就是封装了sql的id以及sql具体内容的集合类

第五步 创建数据源UnPooledDataSource类

  • 数据源的作用就是提供一个数据库连接对象connection,数据源要求实现javax.sql.DataSource接口,这个接口是jdk自带的,所以不需要自己再去定义,事务管理器接口就需要自己定义
  • 数据源所需要的driver,url,password,username等数据,来自于SqlSessionFactoryBuilder的getDataSource方法
    • 即dataSource = new UnPooledDataSource(driver, url, username, password);
    • 这些数据依然来自于xml文件解析
  • 此外还可以创建多个数据源(PooledDataSource和JNDIDataSource),这里省略了

第六步 创建事务管理器Transaction接口

  • 和数据源接口不同,事务管理器接口需要自己定义
  • 事务管理器规定了事务的基本控制方法,比如事务的提交,关闭,回滚以及获取数据库连接对象

第七步 创建JDBCTransaction事务管理器实现类

  • 事务管理器的具体实现类,同样可以具有多个实现类(比如ManagedTransaction类,这里依然省略了),由用户进行xml文件配置时,决定使用哪个事务管理器
  • 事务管理器的作用是根据提供数据源对象,获取到数据源对象提供的数据库连接对象connection,进而进行事务管理
  • 事务管理器需要两个参数 public JDBCTransaction(DataSource dataSource, boolean autoCommit),这里第二个参数并没有什么实际意义,所以写死了.第一个参数由SqlSessionFactoryBuilder的getTransaction()方法提供
    • transaction = new JDBCTransaction(dataSource, false);//默认是开启事务的，需要手动提交

第八步 创建SqlSessionFactory类

  • 前面的准备工作已经可以让SqlSessionFactoryBuilder拥有足够的参数,返回一个SqlSessionFactory对象了
  • 查看测试类中SqlSession sqlSession = sqlSessionFactory.openSession();可知SqlSessionFactory应该有一个openSession()方法能够获取到一个SqlSession对象
  • 再查看测试类中sqlSession.selectOne(“user.selectUser”, “2”),知sqlSession对象,应该能够根据到sql语句操作数据库,所以sqlSession需要数据源以及sql封装对象(MappedStatement)
  • 至此就给了sqlSession对象足够多的数据

第九步 创建SqlSession类

  • 根据测试类中
    • sqlSession.selectOne(“user.selectUser”, “2”)
    • sqlSession.insert(“user.insertUser”, user)
    • sqlSession.commit();
    • sqlSession.close();
  • 分析出应具有这些方法,前两个方法都需要进行xml文件解析,获取到sql的id,然后从Sql封装对象的Map集合(mappedStatements)中拿到对应的sql信息(mappedStatement),还要再对参数进行获取,通过替换sql语句中#{}进行sql拼串操作,获取到完整的sql语句后,通过事务管理器拿到数据库操作对象connection进行jdbc操作
  • 此外select操作会拿到数据库里的一些数据(键值对),还要通过反射机制获取到对应(resultType)的类对象,之后创建并调用对应的set方法进行赋值,返回赋值好的对象,使用者进行数据类型强转即可
